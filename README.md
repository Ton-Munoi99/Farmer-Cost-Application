# Rice Cost Manager

Mobile-first PWA for Thai rice farmers to record farm costs, daily revenue, profit/loss, season history, reports, and local backup/import.

Live app: https://farmer-cost-application.netlify.app

## Features

- Installable PWA with offline support
- Local-first data storage with `localStorage`
- Cost and revenue entries by date/category
- Edit/delete entries
- Season summary and history
- Multiple editable seasons with safe season switching
- Charts/report screen
- Export `.xlsx` reports
- Backup/import `.json` data for moving between devices
- Monthly backup reminder with last-backup date
- Thai/English UI toggle

## Development

```powershell
npm install
npm run dev
```

## Production Build

```powershell
npm run build
npm run preview
```

The static production output is generated in `dist/`.

## Deploy

Netlify is configured through `netlify.toml`.

```toml
[build]
  command = "npm run build"
  publish = "dist"
```

Pushing to `main` on GitHub triggers Netlify auto deploy.

## PWA Notes

- `manifest.json` defines install metadata and icons.
- Production `sw.js` is generated by `vite.config.js` during build.
- The generated service worker precaches the built files and includes a content hash in `CACHE_VERSION`, so app updates refresh the offline cache.
- HTTPS is required for install/offline behavior outside localhost.

## Data Model

Main persisted key:

```text
ricecost_v1
```

Stored data:

- `farm`
- `costEntries`
- `revenue.entries`
- `history`
- `tweaks`

Data stays on the user's device unless they export a backup file. Backup/import uses plain `.json` so users can move data between phone and computer without login or cloud sync.
